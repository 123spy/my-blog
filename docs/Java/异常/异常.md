java的一场体系结构，可以分为两类

error的错误

exception异常



error代表的是，计算机出现了无法处理的JVM级别错误，比如：

栈溢出

内存溢出



exception代表的是，程序导致的或者外在因素比如断网等导致的问题。



对于exception异常，主要分为了两类

运行时异常：

继承自RunTimeException

**特点**：**编译器不强制要求处理**。你可以不写 try-catch，编译也能通过。

**本质**：通常是**逻辑错误**。比如数组越界、空指针。

**常见例子**：

- `NullPointerException` (空指针异常)
- `ArithmeticException` (算术异常，如 10 / 0)
- `ArrayIndexOutOfBoundsException` (数组下标越界)
- `ClassCastException` (类型转换异常)



编译时异常

除了RunTimeException之外的Exception

**特点**：**编译器强制要求处理**。如果你不处理（捕获或抛出），代码**连编译都通不过**。

**本质**：通常是外部资源问题，必须预案。

**常见例子**：

- IOException（输入输出异常）
- FileNotFoundException（文件未找到异常）
- SQLException（数据库异常）
- ClassNotFoundException（类未找到异常）
- InterruptedException（线程中断异常）
